Overview

This guide is for version 1.1 of the MultimediaGallery.

The version number is listed in the DevTools panel, which is only viewable when the gallery is running in devMode.

Installation

Mac OS

  1. Open the node-v11.2.0.pkg file. Follow the instructions to install Node.
  2. Copy the MultimediaGallery application using the Finder.
  3. Open the Applications folder alias. This should open the Applications folder on your computer.
  4. Paste the MultimediaGallery application into the Applications folder using the Finder.
  5. Open the MultimediaGallery application. The gallery should load and open in full screen kiosk mode.
Quitting

To quit the program with a keyboard, enter the 4-digit pin or use the Command+Q keyboard shortcut. If no keyboard is available, navigate to the Credits popup and hold three fingers on the logo image for 3 seconds.

Making Changes

To make changes to the Gallery, you will need access to the content folder. This folder is hidden by default. To access it:

  1. Open the Applications folder.
  2. Right-click on the MultimediaGallery application and choose Show Package Contents.
  3. Open the public folder.
  4. Open the content folder.

Windows

  1. Open the node-v11.2.0-x64.msi file. Follow the instructions to install Node. Use the default settings.
  2. Copy the MultimediaGallery folder to the C: drive so it is located at C:/MultimediaGallery.
  3. Open the MultimediaGallery folder on the C: drive.
  4. Open the Package folder and the Scripts folder.
  5. Run the InstallModules file. This may take a while.
  6. Return to the MultimediaGallery folder.
  7. Run the StartMultimediaGallery file.
Disabling Windows 10 Keyboard Shortcuts
  1. Open the Local Group Policy Editor (Windows 10 Pro or Windows 10 Educational)
  2. Go to User Configuration > Administrative Templates > System > CTRL + ALT + Del Options > Remove Lock Computer
    Set to Enabled
  3. Go to User Configuration > Administrative Templates > System > CTRL + ALT + Del Options > Remove Task Manager
    Set to Enabled
  4. Go to User Configuration > Administrative Templates > System > CTRL + ALT + Del Options > Remove Change Password
    Set to Enabled
  5. Go to User Configuration > Administrative Templates > System > CTRL + ALT + Del Options > Remove Logoff
    Set to Enabled
  6. Go to Computer Configuration > Administrative Templates > System > Logon > Hide entry points for Fast User Switching
    Set to Enabled
  7. Go to Computer Configuration > Administrative Templates > System > Device Installation > Device Installation Restrictions > Prevent installation of devices not described by other policy settings
    Set to Enabled
  8. Go to Computer Configuration > Administrative Templates > System > Logon > Turn off app notifications on the lock screen
    Set to Enabled
Disabling Windows Explorer and Edge Gestures

Run the StartMultimediaGalleryDisableExplorer file. Note: this method will turn off Windows Explorer completely.

Alternative Method to Disable Edge Gestures

  1. Open the Local Group Policy Editor (Windows 10 Pro or Windows 10 Educational)
  2. Go to Computer Configuration > Administrative Templates > Windows Components > Edge UI
    Set to Disable
    3. Quitting

    Making Changes

Folder Structure

All content, media, settings and log files live inside the content folder. See Installation above for steps to locate this folder.

Inside the content folder, you can organize sub-folders any way that makes sense to you.

Files & Folders

Configuration

The configuration.xml file contains all configuration settings for the MultimediaGallery. It is required.

Required Tags

The XML file needs to have a <main> tag. All other tags are nested inside it. The order of required tags does not matter.

Tag Description Parent Required
<main> Container for all other tags. none yes
<settings> Global settings for development mode and timeout duration. <main> yes
<home> Settings and content for the attract screen. <main> yes
<menu> Settings and content for the main menu screen. <main> yes
<toc> Content for the Table of Contents popup. <main> yes
<credits> Content for the Credits popup. <main> yes
<help> Path to the video or image that displays in the Help popup. <main> yes

Optional Tags

There are currently six optional tags. These are for the different media types.

These tags can be placed directly inside the <main> tag or nested inside a <map> tag. If they are nested inside <main>, they will appear as buttons on the main menu. If they are nested in a <map>, they will appear as buttons on that map's carousel. The order here determines the order they will appear on screen.

Note: <map> tags can be nested inside other <map> tags up to three levels deep.

Tag Description Parent Required
<video> video <main> or <map> no
<map> map <main> or <map> no
<gallery> image gallery <main> or <map> no
<panorama> 360 degree panorama <main> or <map> no
<html> local HTML module or remote website <main> or <map> no
<object> 3D object <main> or <map> no

Settings

The <settings> tag contains global settings for development mode and timeout duration. It is required.

Attribute Type Description Required
timeout attribute Number of seconds of inactivity required before the kiosk warns and resets the session. yes
timeoutLong attribute Number of seconds when viewing an HTML module before the kiosk warns and resets the session. yes
pin attribute A 4-digit pin that is used to quit the kiosk. User must enter the pin from the Home / Attract screen. If you do not include a pin, the default pin is "1234". no
mapLogo attribute Path to an image file with a logo that will display on the Maps screen. no
headingFont attribute Path to a font in ttf or woff format that will override the default font for headings throughout the interface. no
copyFont attribute Path to a font in ttf or woff format that will override the default font for copy throughout the interface. no
devMode attribute If devMode="true", the kiosk will start in development mode showing the Log Viewer and Dev Tools panels. no
hideMediaIndex attribute If hideMediaIndex="true", the Media Index button will be hidden. no
hideMainMenu attribute If hideMainMenu="true", the Main Menu will be hidden. When the user touches the attract screen, the kiosk will automatically open the first media element. no

Example


<settings kioskId="kauai-museum-1234" pin="2468" timeout="90" timeoutLong="900" devMode="true" headingFont="fonts/Candara.ttf" mapLogo="images/logo.png" hideMediaIndex="true"></settings>

Help

The <help> tag contains the path to a help video or image. It is required. The video should be in mp4 format. If an image is used, it should be in png or jpg format.

Captions

If the folder contains a .vtt caption file with the same name, it will be used to display closed captions. For example, if the video file is named help.mp4, the caption file should be named help.vtt.

Attribute Type Description Required
file attribute Path to the video in mp4 format or the image in png or jpg format. yes

Video Example


<help file="videos/help.mp4"></help>

Image Example


<help file="images/help.png"></help>

Map

The <map> tag contains settings for a map module.

Nesting

Each map module can have up to 18 nested child modules. For more information on the nesting structure, please review the Configuration section of this guide.

Map Tile Creation

Use Ideum's MapCreationApplication to download map tiles.

  1. Select Map Source: Mapbox
  2. Select Map Style: streets or satellite-streets
  3. Choose the Bounding Box setting.
  4. Enter longitude and latitude coordinates.
  5. Choose a zoom level. Try to pick a zoom level that does not exceed 300 maptiles.
  6. Check the box that says For Testing Purposes Save Entire Map as Single Image. Leave the name field blank.
  7. Click Go and wait for the files to download.
  8. Important: Find the FullMapImage.jpg file. If the width is larger than 3000 pixels, use a tool like Photoshop to resize it so the width is 3000 pixels. Keep the same aspect ratio.
  9. Copy all map files into a folder such as /content/maps/kauai/. Make sure to include the tile images, the FullMapImage.jpg image and the MapImageLayout.json file.

Map Configuration

Tag/Attribute Type Description Required
<title> tag The title of the map module. yes
<description> tag A brief description of the map module. yes
thumb attribute Thumbnail image for the map module. Recommended size is approximately 800 by 600. Since thumbnails may get cropped, keep the focal point of the image in the center. yes
folder attribute Path to the folder containing the map images and MapImageLayout.json file. It is not necessary to include the content folder in the path. yes
mapX attribute A positive or negative number indicating the starting horizontal location of the map. The default value is 0 which will center the map horizontally. Use the Dev Tools to preview mapX and mapY coordinates. Note: this is not used for the web-based version of the MMG. no
mapY attribute A positive or negative number indicating the starting vertical location of the map. The default value is 0 which will center the map vertically. Use the Dev Tools to preview mapX and mapY coordinates. Note: this is not used for the web-based version of the MMG. no
lat attribute Latitude coordinate for the center of the map. Note: this is only used for the web-based version of the MMG. Use the Dev Tools to preview lat and long coordinates. no
long attribute Longitude coordinate for the center of the map. Note: this is only used for the web-based version of the MMG. Use the Dev Tools to preview lat and long coordinates. no
zoom attribute A number between 1 and 10. The default value for the local version of the MMG is 1. This indicates the map should start zoomed out as far as possible. Use the Dev Tools to preview zoom values. Note: zoom values may be different for the web-based version of the MMG. no
maxZoom attribute A number between 1 and 10. The default value is 10. Use the Dev Tools to preview zoom values. Note: maxZoom is currently not applicable to the web-based version of the MMG. no
pinX attribute A positive or negative number indicating the horizontal location of the map pin on the parent map. This is only used if the map is nested within another map. Use the Dev Tools to preview pinX and pinY coordinates. Note: this is not used for the web-based version of the MMG. no
pinY attribute A positive or negative number indicating the vertical location of the map pin on the parent map. This is only used if the map is nested within another map. Use the Dev Tools to preview pinX and pinY coordinates. Note: this is not used for the web-based version of the MMG. no
switchPoint attribute A number between 1 and 10. This number indicates at which zoom value the map should switch from the lower resolution full map image to the high resolution map tiles. The default value is 3. This is only used for performance fine-tuning. Use the Dev Tools to preview different switchPoint values. Note: this is not used for the web-based version of the MMG. no
showTitle attribute If showTitle="true", the title of the map will display in large text centered over the map. This can be useful for maps that do not have any pins. Note: this is not currently available in the web-based version of the MMG. no
hideZoomButtons attribute Use hideZoomButtons="true" to hide the pan and zoom buttons on the map. no
topics attribute A comma separated list of one or more topics. Topics automatically genrate the Topic filter buttons on the Table of Contents popup. no

topics attribute A comma separated list of one or more topics. Topics automatically genrate the Topic filter buttons on the Table of Contents popup. no

Panorama

The <panorama> tag contains settings for a panorama.

All files for a single panorama should be kept in one folder.

Panorama Setup

  1. Use the Marizipano Tool to create a Panorama.
  2. Copy all files into a sub-folder in the content folder such as content/panoramas/lava/. This should include 4 files: data.js, index.html, index.js, style.css and 3 folders: img, tiles, vendor.
  3. Open index.html.
  4. On line 10, find: <link rel="stylesheet" href="style.css">
  5. Underneath it, add: <link rel="stylesheet" href="../../../panorama.css">
  6. Save the file.

3D Object

The <object> tag contains settings for a three dimensional object.

Objects should be in obj and mtl format. Textures should be in png or jpg format. All files should be placed in the same folder and have the same name. For example, a 3D object called bowl will have three files: bowl.obj, bowl.mtl, and bowl.jpg. Texture files should be 512 pixels by 512 pixels.

Attribute Type Description Required
<title> tag The title of the HTML module. yes
<description> tag A brief description of the HTML module. yes
thumb attribute Thumbnail image for the 3D object. Recommended size is approximately 800 by 600. Since thumbnails may get cropped, keep the focal point of the image in the center. yes
folder attribute Path to the folder containing the 3D object files. It is not necessary to include the content folder in the path. yes
file attribute The name of the object file. Do not include file extension. yes
offsetZ attribute A value indicating the default zoom value of the object. The default value is 0. no
pinX attribute A positive or negative number indicating the horizontal location of the map pin on the parent map. This is only used if the media item is nested within a map. Use the Dev Tools to preview pinX and pinY coordinates. no
pinY attribute A positive or negative number indicating the vertical location of the map pin on the parent map. This is only used if the media item is nested within a map. Use the Dev Tools to preview pinX and pinY coordinates. no
topics attribute A comma separated list of one or more topics. Topics automatically genrate the Topic filter buttons on the Table of Contents popup. no
hideFromIndex attribute Use hideFromIndex="true" to indicate that this item should not show up in the Media Index. no

Example


<object thumb="images/bowl.png" folder="objects/" file="model" topics="artifacts" pinX="1048" pinY="397" zOffset="-1.5">

Performance

Following are recommendations to improve performance.

Overall

Use a machine with at least 16GB of RAM.

Maps

Check the dimensions of the FullMapImage.jpg. Make sure the width is 3000 pixels or smaller. Try using a smaller image.

User fewer map tiles. Maps with fewer than 200 tiles will perform best.

Adjust the map switchPoint. If your map has fewer tiles, try setting this number to a lower value between 1 and 2.

Table of Contents

Keep thumbnail images smaller - images that are 800 by 600 pixels or less will load faster than large images.

Image Compression

You may notice improvements to performance by compressing images. This is especially true for large images such as those used for the full map images.

The ImageOptim Mac application is free and can provide significant file size savings for jpg and png files. For more information, see the website: www.imageoptim.com.
